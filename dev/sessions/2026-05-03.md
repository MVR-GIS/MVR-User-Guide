    # Load a repo into Foundry (ZIP import)
    
    ## Objective
    
    Bring an existing GitHub repository into a Foundry Code Repository using a ZIP download/import workflow, so you can run and version your work inside the Foundry enclave.
    
    This chapter is a prerequisite for:
    
    - `fnd-threads.qmd` (Initiate a reproducible Threads workflow)
    
    ## Prerequisites
    
    Before you start, complete:
    
    - `fnd-getting-started.qmd` (login, create a project, tagging for discovery, groups/permissions, etc.)
    
    You also need:
    
    - access to create a Code Repository in your Foundry Project
    - access to the GitHub repository you will import
    
    ## Which repo should I import?
    
    For USACE data science work, we recommend starting from the approved lightweight Python analysis template described in:
    
    - `repo-create-generic-python-repo.qmd`
    
    That chapter’s workflow creates a new GitHub repository based on:
    
    - Template repo: `MVR-GIS/repo-generic-python`
    
    In Foundry, you will typically import *your new repo* (created from the template), not the template itself.
    
    ## Key steps (overview)
    
    1. Create your analysis repo in GitHub from the approved template (external step).
    2. Download a ZIP of the repo from GitHub (“big green code button”).
    3. Create (or open) a Foundry Code Repository in your Foundry Project.
    4. Import/upload the ZIP contents into the Foundry Code Repository.
    5. Verify the repo structure appears as expected.
    
    ## Step 1 — Create your analysis repo in GitHub (outside Foundry)
    
    Follow:
    
    - `repo-create-generic-python-repo.qmd`
    
    When you’re done, you should have a GitHub repository you own/control (example: `MVR-GIS/my-analysis-project`).
    
    ## Step 2 — Download the repo ZIP from GitHub
    
    1. Open your analysis repo in GitHub.
    2. Click **Code** (green button).
    3. Click **Download ZIP**.
    4. Save the ZIP somewhere you can access for import to Foundry.
    
    Notes:
    
    - If your Foundry enclave environment cannot access GitHub directly, you may need to move the ZIP through an approved transfer mechanism. Follow your organization’s policy.
    - Prefer a tagged release ZIP if you need stronger provenance. Otherwise, record the commit SHA you downloaded for traceability.
    
    ## Step 3 — Create/open a Foundry Code Repository
    
    In Foundry (inside the correct Project):
    
    1. Create a new Code Repository (or open an existing one).
    2. Name it consistently with the GitHub repo you’re importing (recommended).
    3. Confirm the Code Repo is in the correct Project and has the right permissions/groups (per `fnd-getting-started.qmd`).
    
    ## Step 4 — Import the ZIP contents into the Foundry Code Repository
    
    Import mechanics vary by Foundry configuration (UI features and permissions). Use the import option available in your environment, such as:
    
    - Upload/import from a ZIP archive
    - Upload folder contents via the UI
    - A “bootstrap” repository import step (if configured)
    
    Required outcome:
    
    - The Code Repository file tree inside Foundry matches the GitHub repository contents at the commit you downloaded.
    
    ## Step 5 — Verify repository structure in Foundry
    
    At minimum, verify you see the expected top-level folders/files (as defined by your analysis repo), such as:
    
    - `notebooks/`
    - `src/`
    - `data/` (if your policy allows storing local data in-repo; often data should be in Foundry Datasets instead)
    - `outputs/`
    - `README.md`
    - `environment.yml` (if applicable)
    
    ## Offline / no-internet guidance (Foundry enclave)
    
    Assume no external network access from runtime environments unless your platform team confirms otherwise.
    
    Recommendations:
    
    - Do not rely on `pip install ...` or `conda env create ...` fetching packages from the internet during normal execution.
    - Prefer platform-provided Python environments or curated package sets where available.
    - If dependency installation is required, coordinate with your Foundry administrators on the approved mechanism (mirrors, curated channels, base images, etc.).
    
    ## Definition of done
    
    You are ready to proceed when:
    
    - the GitHub repo ZIP has been imported into a Foundry Code Repository
    - you can browse the repo file tree in Foundry
    - you recorded provenance:
      - GitHub repo URL
      - commit SHA (or tag/release) corresponding to the ZIP you imported
    - permissions/groups are set so collaborators can access the Code Repo (as intended)

    # Initiate a reproducible Threads workflow
    
    ## Objective
    
    Initiate a reproducible workflow for capturing a Threads chat session history and saving outputs in a reviewable, auditable way inside a Foundry enclave (assume offline / no-internet execution).
    
    This chapter focuses on *workflow and structure*, not on account onboarding or Foundry project setup.
    
    ## Prerequisites
    
    Complete these first:
    
    - `fnd-getting-started.qmd`
      - login
      - create a Project
      - tagging for discovery
      - groups/permissions
      - any required platform conventions
    
    - `fnd-load-repo.qmd`
      - import your GitHub analysis repository into a Foundry Code Repository using ZIP import
    
    You should also already have a GitHub analysis repo created from:
    
    - `repo-create-generic-python-repo.qmd`
    
    ## What is a “reproducible Threads session” in this context?
    
    A reproducible Threads session should leave an audit trail that includes:
    
    - who/what: analyst, date/time window, project context
    - inputs: the prompt(s), any system/context constraints, and referenced datasets
    - configuration: model/tool settings as allowed by your environment
    - outputs: generated text, extracted entities/tables, files produced
    - provenance: dataset RIDs/paths (or equivalent identifiers), code version/commit, and run identifier
    
    Your goal is that a reviewer can answer:
    
    - “What question was asked?”
    - “What data and code were used?”
    - “What did the assistant produce?”
    - “Where are the saved artifacts?”
    
    ## Key steps (overview)
    
    1. Ensure your Foundry Code Repo is imported and ready.
    2. Create a standard session folder for the Threads run.
    3. Capture session metadata (who/what/why, dataset references, config).
    4. Capture the session transcript (prompts + responses) in a stable format.
    5. Save outputs (and any derived artifacts) with deterministic naming.
    6. Commit changes to the Code Repo (or save to an approved dataset/artifact location) to preserve an audit trail.
    
    ## Recommended repository structure for Threads sessions
    
    Use a stable location inside the repo for session artifacts. Recommended pattern (create if missing):
    
    - `threads/`
      - `sessions/`
        - `<YYYY-MM-DD>_<short-topic-slug>/`
          - `README.md`
          - `metadata.yml`
          - `transcript.md`
          - `outputs/`
    
    Notes:
    
    - Keep “session history” in plain text formats (`.md`, `.yml`, `.json`) to maximize portability and diffability.
    - Avoid proprietary binary formats for primary records.
    - If outputs are large, store summaries in-repo and write full outputs to an approved Foundry dataset/artifact location (recording identifiers in `metadata.yml`).
    
    ## Step 1 — Create a new session folder
    
    Create a new folder under:
    
    - `threads/sessions/`
    
    Naming convention:
    
    - `YYYY-MM-DD_short-topic-slug`
    
    Example:
    
    - `2026-05-03_real-estate-permits-qaqc`
    
    ## Step 2 — Record metadata first (before you chat)
    
    Create `metadata.yml` with, at minimum:
    
    - session name
    - date
    - author
    - purpose / question
    - datasets referenced (identifiers, not copies)
    - constraints (offline/no-internet, sensitive handling notes)
    - code provenance (commit SHA if available)
    
    Recommendation:
    
    - include a section that explicitly states what is *out of scope* or what was *not* done (helps reviewers).
    
    ## Step 3 — Run the Threads session
    
    Conduct the Threads chat session following your organization’s policy on:
    
    - what data can be pasted into chat
    - how sensitive data is handled
    - what model/tooling is approved
    
    During the session, keep a scratchpad of:
    
    - each prompt you send
    - each key response you receive
    - any action items or decisions
    
    ## Step 4 — Save the transcript (stable + reviewable)
    
    Create `transcript.md` and capture:
    
    - the prompts (verbatim)
    - the responses (verbatim)
    
    If your platform provides an export function, prefer that over manual copy/paste, but ensure:
    
    - the format is plain text / markdown
    - timestamps (if present) are consistent and not misleading
    - you remove/redact sensitive data per policy
    
    ## Step 5 — Save outputs (and keep them traceable)
    
    Put any derived artifacts in `outputs/`, such as:
    
    - a cleaned “final answer” version of the response
    - extracted tables as CSV
    - checklists or SOP steps as markdown
    
    Naming guidance:
    
    - keep names deterministic and descriptive
    - avoid spaces; use hyphens
    
    Examples:
    
    - `outputs/final-summary.md`
    - `outputs/extracted-entities.csv`
    - `outputs/recommended-workflow-checklist.md`
    
    ## Step 6 — Preserve an audit trail
    
    Minimum viable audit trail:
    
    - `metadata.yml` exists and references:
      - datasets used (identifiers)
      - where outputs are stored
      - the repo version / commit (or a note that this is not available)
    - `transcript.md` is present
    - outputs are saved under `outputs/`
    
    Preferred:
    
    - commit the session folder to the Foundry Code Repo with a clear commit message, or
    - publish the artifacts to an approved Foundry location and link to them from the session README/metadata
    
    ## Offline / no-internet constraints
    
    Design your workflow so it does not require:
    
    - downloading packages at run time
    - calling external APIs
    - embedding links as “required steps” unless internal and accessible
    
    If you reference external documentation, treat it as “background reading” rather than a dependency for completing the workflow.
    
    ## Troubleshooting
    
    ### I can’t import the repo ZIP into Foundry
    
    - Re-check `fnd-load-repo.qmd`
    - Confirm you have permissions to create or modify a Code Repository in the target Project
    - Confirm you are using an approved transfer path for moving the ZIP into the enclave
    
    ### I’m not sure what I’m allowed to paste into Threads
    
    - Stop and confirm policy with your platform/data governance point of contact
    - Prefer referencing datasets by identifier rather than pasting raw records
    
    ### Outputs are too large to store in the repo
    
    - Store summaries and metadata in-repo
    - Store large artifacts in an approved Foundry dataset/artifact location
    - Record exact identifiers/paths in `metadata.yml`
    
    ## Definition of done
    
    You are done when:
    
    - a new session folder exists under `threads/sessions/`
    - `metadata.yml` exists and includes:
      - purpose/question
      - dataset identifiers (if any)
      - constraints (offline/no-internet)
      - where outputs were saved
    - `transcript.md` contains a complete prompt/response record (redacted as needed)
    - outputs are saved and referenced
    - the work is preserved in a way a reviewer can retrieve later (commit and/or published artifacts)

    def extract_foundry_threads_chat(

    def _render_markdown(export: Dict[str, Any], export_sha256: str) -> str:
        meta = export.get("metadata", {}) if isinstance(export, dict) else {}
    
        title = _as_str(meta.get("title"))
        created_at = _as_str(meta.get("createdAt"))
        last_updated_at = _as_str(meta.get("lastUpdatedAt"))
        user_id = _as_str(meta.get("userId"))
    
        messages = _extract_messages(export)
    
        # participant enrichment (preserve username + model)
        user_display = _first_nonempty([m.display_name for m in messages if m.role == "user"])
        model_display = _first_nonempty([m.display_name for m in messages if m.role == "assistant"])
    
        counts_total = len(messages)
        counts_user = sum(1 for m in messages if m.role == "user")
        counts_assistant = sum(1 for m in messages if m.role == "assistant")
    
        front_matter = {
            "title": title,
            "created_at": created_at,
            "last_updated_at": last_updated_at,
            "foundry_user_id": user_id,
            "user_display_name": user_display,
            "model_display_name": model_display,
            "export_sha256": export_sha256,
            "messages_total": counts_total,
            "messages_sent": counts_user,
            "messages_received": counts_assistant,
            "source": {
                "platform": "foundry",
                "export_file": "threads_export.json",
                "format": "foundry_threads_export",
            },
        }
    
        yaml_text = yaml.safe_dump(front_matter, sort_keys=False, allow_unicode=True).strip()
    
        lines: List[str] = []
        lines.append("---")
        lines.append(yaml_text)
        lines.append("---")
        lines.append("")
        lines.append("# Threads session transcript")
        lines.append("")
    
        if title:
            lines.append(f"## Title")
            lines.append("")
            lines.append(title)
            lines.append("")
    
        # Conversation
        lines.append("## Conversation")
        lines.append("")
    
        for i, m in enumerate(messages, start=1):
            role_label = "User" if m.role == "user" else "Assistant"
            name_part = f" — {m.display_name}" if m.display_name else ""
            lines.append(f"### {i}. {role_label}{name_part}")
            lines.append("")
            lines.append(m.content.strip() if m.content else "")
            lines.append("")

    @dataclass(frozen=True)
    class Message:
        role: str               # "user" | "assistant"
        display_name: str
        content: str
    
    
    def _extract_messages(export: Dict[str, Any]) -> List[Message]:
        conv = export.get("conversation", [])
        if not isinstance(conv, list):
            return []
    
        out: List[Message] = []
    
        for item in conv:
            if not isinstance(item, dict) or len(item) != 1:
                continue
    
            key = next(iter(item.keys()))
            payload = item.get(key)
            if not isinstance(payload, dict):

    def extract_foundry_threads_chat(
        export_path: Optional[str | Path] = None,
        session_date: Optional[str | dt.date] = None,
        topic: Optional[str] = None,
        sessions_dir: str | Path = "dev/sessions",
        backup: bool = True,
        quiet: bool = False,
        downloads_dir: Optional[str | Path] = None,
    ) -> Path:
        """
        Extract Foundry Threads export JSON into a reproducible session transcript.
    
        Writes/updates:
          - dev/sessions/YYYY-MM-DD[_topic].md
          - dev/sessions/.raw/YYYY-MM-DD[_topic].threads_export.json
    
        Backup behavior (modeled after reproducibleai::extract_copilot_chat):
          - if the markdown exists and backup=True, copy it to:
              dev/sessions/.backups/<stem>_backup_YYYYmmdd_HHMMSS.md
          - then overwrite the markdown in place
    
        Defaults:
          - export_path defaults to Downloads/threads_export.json
          - session_date defaults to export metadata.createdAt (RFC1123) else today
          - topic defaults to None (date-only naming, reproducibleai pattern)
        """
        sessions_dir = Path(sessions_dir)
        _ensure_dir(sessions_dir, quiet=quiet)
    
        export_path = _resolve_export_path(export_path=export_path, downloads_dir=downloads_dir)
        if not export_path.exists():
            raise FileNotFoundError(f"Export JSON not found: {export_path}")
    
        export_text = export_path.read_text(encoding="utf-8")
        export = json.loads(export_text)
    
        date_str = _resolve_session_date_str(session_date=session_date, export=export)
        topic_clean = _sanitize_topic(topic) if topic else None
    
        stem = date_str if topic_clean is None else f"{date_str}_{topic_clean}"
        md_path = sessions_dir / f"{stem}.md"
    
        raw_dir = sessions_dir / ".raw"
        _ensure_dir(raw_dir, quiet=quiet)
        raw_path = raw_dir / f"{stem}.threads_export.json"
    
        backups_dir = sessions_dir / ".backups"
        if backup:
            _ensure_dir(backups_dir, quiet=quiet)
    
        # Backup existing markdown (COPY, not move)
        if md_path.exists() and backup:
            backup_path = _make_backup_path(backups_dir, md_path.name)
            shutil.copy2(md_path, backup_path)
            if not quiet:
                print(f"Backed up existing session to: {backup_path}")
    
        # Write raw export (committed)
        raw_path.write_text(export_text, encoding="utf-8")
        if not quiet:
            print(f"Wrote raw export: {raw_path}")
    
        export_sha256 = _sha256_text(export_text)
    
        # Write transcript markdown (committed)
        transcript_md = _render_markdown(export=export, export_sha256=export_sha256)
        md_path.write_text(transcript_md, encoding="utf-8")
        if not quiet:
            print(f"Wrote transcript: {md_path}")
    
        return md_path

    def _render_markdown(export: Dict[str, Any], export_sha256: str) -> str:
        meta = export.get("metadata", {}) if isinstance(export, dict) else {}
    
        title = _as_str(meta.get("title"))
        created_at = _as_str(meta.get("createdAt"))
        last_updated_at = _as_str(meta.get("lastUpdatedAt"))
        user_id = _as_str(meta.get("userId"))
    
        messages = _extract_messages(export)
    
        user_display = _first_nonempty([m.display_name for m in messages if m.role == "user"])
        model_display = _first_nonempty([m.display_name for m in messages if m.role == "assistant"])
    
        counts_total = len(messages)
        counts_user = sum(1 for m in messages if m.role == "user")
        counts_assistant = sum(1 for m in messages if m.role == "assistant")
    
        front_matter = {
            "title": title,
            "created_at": created_at,
            "last_updated_at": last_updated_at,
            "foundry_user_id": user_id,
            "user_display_name": user_display,
            "model_display_name": model_display,
            "export_sha256": export_sha256,
            "messages_total": counts_total,
            "messages_sent": counts_user,
            "messages_received": counts_assistant,
            "source": {
                "platform": "foundry",
                "export_file": "threads_export.json",
                "format": "foundry_threads_export",
            },
        }
    
        yaml_text = yaml.safe_dump(front_matter, sort_keys=False, allow_unicode=True).strip()
    
        lines: List[str] = []
        lines.append("---")
        lines.append(yaml_text)
        lines.append("---")
        lines.append("")
        lines.append("# Threads session transcript")
        lines.append("")

            lines.append("## Title")
            lines.append("")
            lines.append(title)
            lines.append("")
    
        lines.append("## Conversation")
        lines.append("")

            lines.append("")
            lines.append(m.content.strip() if m.content else "")
            lines.append("")

    def extract_foundry_threads_chat(
        export_path: Optional[str | Path] = None,
        session_date: Optional[str | dt.date] = None,
        topic: Optional[str] = None,
        sessions_dir: str | Path = "dev/sessions",
        backup: bool = True,
        quiet: bool = False,
        downloads_dir: Optional[str | Path] = None,
    ) -> Path:
        """
        Extract a Foundry Threads chat export JSON file into a reproducible, human-readable
        markdown transcript with YAML front matter, and preserve the raw export JSON.
    
        This function is modeled after the behavior of `{reproducibleai}::extract_copilot_chat()`:
    
        - Minimal required arguments with sensible defaults
        - "Save often" workflow: re-running updates the *same* derived artifact
        - On overwrite, creates timestamped backups (default: enabled)
    
        Inputs
        ------
        export_path:
            Path to the Foundry Threads export JSON file (typically `threads_export.json`).
            If None (default), uses:
              - Windows: %USERPROFILE%\\Downloads\\threads_export.json
              - Other OS: ~/Downloads/threads_export.json
    
        session_date:
            Session date used in output naming.
            If None (default), attempts to parse `metadata.createdAt` from the export
            (RFC1123 format like "Sun, 03 May 2026 15:53:07 GMT"). If parsing fails,
            falls back to today's date.
    
            You may also pass:
              - a datetime.date object, or
              - a string "YYYY-MM-DD"
    
        topic:
            Optional topic suffix for multi-topic days. Mirrors `{reproducibleai}`:
            if None, the output is date-only (YYYY-MM-DD.md). If provided, the output
            is YYYY-MM-DD_<topic>.md, after sanitization.
    
        sessions_dir:
            Directory for session artifacts (default: "dev/sessions").
    
        backup:
            If True (default), when overwriting an existing session transcript,
            copy it to `dev/sessions/.backups/` with a timestamp suffix before
            writing the new transcript.
    
        quiet:
            If True, suppress informational messages.
    
        downloads_dir:
            Advanced override for the downloads directory used when export_path is None.
    
        Outputs (written)
        -----------------
        Committed artifacts (Option A):
          - dev/sessions/YYYY-MM-DD[_topic].md
              Human-readable transcript with YAML front matter.
          - dev/sessions/.raw/YYYY-MM-DD[_topic].threads_export.json
              Raw export preserved for audit and for re-deriving the transcript.
    
        Gitignored artifacts:
          - dev/sessions/.backups/
              Timestamped backups of the transcript created on overwrite.
    
        Returns
        -------
        Path
            The path to the written transcript markdown file.
    
        Usage (typical)
        ---------------
        1) Export Threads chat JSON from Foundry UI to `threads_export.json`
           (OS default Downloads directory).
        2) From repo root, run:
    
           python -m tools.foundry_threads
    
        3) Commit:
           - dev/sessions/YYYY-MM-DD*.md
           - dev/sessions/.raw/*.threads_export.json
    
        Notes
        -----
        - This function assumes offline/no-internet friendly operation.
        - The Foundry export format observed includes a `conversation` list with
          objects like {"SENT": {...}} and {"RECEIVED": {...}}.
        - A known export typo is handled: "diplayName" instead of "displayName".
        """
        sessions_dir = Path(sessions_dir)
        _ensure_dir(sessions_dir, quiet=quiet)
    
        export_path = _resolve_export_path(export_path=export_path, downloads_dir=downloads_dir)
        if not export_path.exists():
            raise FileNotFoundError(f"Export JSON not found: {export_path}")
    
        export_text = export_path.read_text(encoding="utf-8")
        export = json.loads(export_text)
    
        date_str = _resolve_session_date_str(session_date=session_date, export=export)
        topic_clean = _sanitize_topic(topic) if topic else None
    
        stem = date_str if topic_clean is None else f"{date_str}_{topic_clean}"
        md_path = sessions_dir / f"{stem}.md"
    
        raw_dir = sessions_dir / ".raw"
        _ensure_dir(raw_dir, quiet=quiet)
        raw_path = raw_dir / f"{stem}.threads_export.json"
    
        backups_dir = sessions_dir / ".backups"
        if backup:
            _ensure_dir(backups_dir, quiet=quiet)
    
        # Backup existing markdown (COPY, not move)
        if md_path.exists() and backup:
            backup_path = _make_backup_path(backups_dir, md_path.name)
            shutil.copy2(md_path, backup_path)
            if not quiet:
                print(f"Backed up existing session to: {backup_path}")
    
        # Write raw export (committed)
        raw_path.write_text(export_text, encoding="utf-8")
        if not quiet:
            print(f"Wrote raw export: {raw_path}")
    
        export_sha256 = _sha256_text(export_text)
    
        # Write transcript markdown (committed)
        transcript_md = _render_markdown(export=export, export_sha256=export_sha256)
        md_path.write_text(transcript_md, encoding="utf-8")
        if not quiet:
            print(f"Wrote transcript: {md_path}")
    
        return md_path
    
    
    @dataclass(frozen=True)
    class Message:
        """
        A normalized message extracted from a Foundry Threads export.
    
        Attributes
        ----------
        role:
            "user" for SENT messages, "assistant" for RECEIVED messages.
        display_name:
            Display name for user or model (if present).
        content:
            Message content as plain text (verbatim from export).
        """
    
        role: str
        display_name: str
        content: str
    
    
    def _render_markdown(export: Dict[str, Any], export_sha256: str) -> str:
        """
        Render a Foundry Threads export as markdown with YAML front matter.
    
        Parameters
        ----------
        export:
            Parsed JSON export (dict).
        export_sha256:
            SHA256 hash of the raw JSON text (for integrity tracking).
    
        Returns
        -------
        str
            Markdown document text ending in a newline.
        """
        meta = export.get("metadata", {}) if isinstance(export, dict) else {}
    
        title = _as_str(meta.get("title"))
        created_at = _as_str(meta.get("createdAt"))
        last_updated_at = _as_str(meta.get("lastUpdatedAt"))
        user_id = _as_str(meta.get("userId"))
    
        messages = _extract_messages(export)
    
        user_display = _first_nonempty([m.display_name for m in messages if m.role == "user"])
        model_display = _first_nonempty([m.display_name for m in messages if m.role == "assistant"])
    
        counts_total = len(messages)
        counts_user = sum(1 for m in messages if m.role == "user")
        counts_assistant = sum(1 for m in messages if m.role == "assistant")
    
        front_matter = {
            "title": title,
            "created_at": created_at,
            "last_updated_at": last_updated_at,
            "foundry_user_id": user_id,
            "user_display_name": user_display,
            "model_display_name": model_display,
            "export_sha256": export_sha256,
            "messages_total": counts_total,
            "messages_sent": counts_user,
            "messages_received": counts_assistant,
            "source": {
                "platform": "foundry",
                "export_file": "threads_export.json",
                "format": "foundry_threads_export",
            },
        }
    
        yaml_text = yaml.safe_dump(front_matter, sort_keys=False, allow_unicode=True).strip()
    
        lines: List[str] = []
        lines.append("---")
        lines.append(yaml_text)
        lines.append("---")
        lines.append("")
        lines.append("# Threads session transcript")
        lines.append("")
    
        if title:
            lines.append("## Title")
            lines.append("")
            lines.append(title)
            lines.append("")
    
        lines.append("## Conversation")
        lines.append("")
    
        for i, m in enumerate(messages, start=1):
            role_label = "User" if m.role == "user" else "Assistant"
            name_part = f" — {m.display_name}" if m.display_name else ""
            lines.append(f"### {i}. {role_label}{name_part}")
            lines.append("")
            lines.append(m.content.strip() if m.content else "")
            lines.append("")
    
        return "\n".join(lines).rstrip() + "\n"
    
    
    def _extract_messages(export: Dict[str, Any]) -> List[Message]:
        """
        Extract normalized messages from the Foundry Threads export structure.
    
        Handles:
        - {"SENT": {...}} -> role="user"
        - {"RECEIVED": {...}} -> role="assistant"
    
        Also handles a known typo in some exports:
        - "diplayName" instead of "displayName" for SENT messages.
        """
        conv = export.get("conversation", [])
        if not isinstance(conv, list):
            return []
    
        out: List[Message] = []
    
        for item in conv:
            if not isinstance(item, dict) or len(item) != 1:
                continue
    
            key = next(iter(item.keys()))
            payload = item.get(key)
            if not isinstance(payload, dict):
                continue
    
            if key == "SENT":
                display = _as_str(payload.get("displayName")) or _as_str(payload.get("diplayName"))
                content = _as_str(payload.get("content"))
                out.append(Message(role="user", display_name=display, content=content))
    
            elif key == "RECEIVED":
                display = _as_str(payload.get("displayName"))
                content = _as_str(payload.get("content"))
                out.append(Message(role="assistant", display_name=display, content=content))
    
        return out
    
    
    def _resolve_export_path(
        export_path: Optional[str | Path],
        downloads_dir: Optional[str | Path],
    ) -> Path:
        """
        Resolve the export JSON path.
    
        If export_path is provided, returns it as a Path.
        Otherwise returns <Downloads>/threads_export.json using OS defaults
        (Windows-first).
        """
        if export_path is not None:
            return Path(export_path)
    
        downloads = Path(downloads_dir) if downloads_dir is not None else _default_downloads_dir()
        return downloads / "threads_export.json"
    
    
    def _default_downloads_dir() -> Path:
        """
        Determine the OS default Downloads directory (Windows-first).
    
        Windows:
          - %USERPROFILE%\\Downloads
    
        Other:
          - ~/Downloads
        """
        if platform.system().lower().startswith("win"):
            userprofile = os.environ.get("USERPROFILE")
            if userprofile:
                return Path(userprofile) / "Downloads"
        return Path.home() / "Downloads"
    
    
    def _resolve_session_date_str(session_date: Optional[str | dt.date], export: Dict[str, Any]) -> str:
        """
        Resolve session date string used in output file naming.
    
        If session_date is None:
          - parse export.metadata.createdAt as RFC1123
          - else fallback to today's date
    
        If session_date is a date:
          - format YYYY-MM-DD
    
        If session_date is a string:
          - must match YYYY-MM-DD
        """
        if session_date is None:
            meta = export.get("metadata", {}) if isinstance(export, dict) else {}
            created = _as_str(meta.get("createdAt"))
            parsed = _try_parse_rfc1123(created) if created else None
            d = parsed.date() if parsed else dt.date.today()
            return d.strftime("%Y-%m-%d")
    
        if isinstance(session_date, dt.date):
            return session_date.strftime("%Y-%m-%d")
    
        s = str(session_date).strip()
        if not re.match(r"^\d{4}-\d{2}-\d{2}$", s):
            raise ValueError("session_date must be a date or 'YYYY-MM-DD'")
        return s
    
    
    def _try_parse_rfc1123(s: str) -> Optional[dt.datetime]:
        """
        Try parsing Foundry's createdAt/lastUpdatedAt format:
          "Sun, 03 May 2026 15:53:07 GMT"
    
        Returns None if parsing fails.
        """
        try:
            return dt.datetime.strptime(s, "%a, %d %b %Y %H:%M:%S GMT")
        except Exception:
            return None
    
    
    def _sanitize_topic(topic: str) -> str:
        """
        Sanitize a topic suffix for use in filenames.
    
        Mirrors `{reproducibleai}` behavior:
          - replace non [a-zA-Z0-9_-] with underscore
          - collapse multiple underscores
          - trim underscores
        """
        topic = topic.strip()
        if not topic:
            return ""
        topic_clean = re.sub(r"[^a-zA-Z0-9_-]", "_", topic)
        topic_clean = re.sub(r"_+", "_", topic_clean).strip("_")
        return topic_clean
    
    
    def _sha256_text(text: str) -> str:
        """Return sha256 hex digest for the provided text."""
        return hashlib.sha256(text.encode("utf-8")).hexdigest()
    
    
    def _ensure_dir(path: Path, quiet: bool = False) -> None:
        """Create directory (recursive) if missing."""
        if not path.exists():
            path.mkdir(parents=True, exist_ok=True)
            if not quiet:
                print(f"Created directory: {path}")
    
    
    def _make_backup_path(backups_dir: Path, original_name: str) -> Path:
        """Create a timestamped backup path for an existing transcript."""
        timestamp = dt.datetime.now().strftime("%Y%m%d_%H%M%S")
        stem = Path(original_name).stem
        return backups_dir / f"{stem}_backup_{timestamp}.md"
    
    
    def _as_str(x: Any) -> str:
        """Coerce value to string; return empty string for None."""
        return "" if x is None else str(x)
    
    
    def _first_nonempty(items: List[str]) -> str:
        """Return the first non-empty string; otherwise empty string."""
        for x in items:
            if x and str(x).strip():
                return str(x).strip()
        return ""
    
    
    def _build_arg_parser() -> argparse.ArgumentParser:
        """Build the CLI argument parser for module execution."""
        p = argparse.ArgumentParser(
            description="Extract Foundry Threads chat export JSON into dev/sessions markdown + raw json (with backups)."
        )
        p.add_argument(
            "--export-path",
            default=None,
            help="Path to threads_export.json. Default: Downloads/threads_export.json",
        )
        p.add_argument(
            "--session-date",
            default=None,
            help="Session date YYYY-MM-DD. Default: parsed from export metadata.createdAt, else today.",
        )
        p.add_argument(
            "--topic",
            default=None,
            help="Optional topic suffix (sanitized). Default: none (date-only filename).",
        )
        p.add_argument(
            "--sessions-dir",
            default="dev/sessions",
            help="Sessions directory. Default: dev/sessions",
        )
        p.add_argument(
            "--no-backup",
            action="store_true",
            help="Disable backups (not recommended).",
        )
        p.add_argument(
            "--quiet",
            action="store_true",
            help="Suppress informational messages.",
        )
        p.add_argument(
            "--downloads-dir",
            default=None,
            help="Override downloads directory (advanced). Default: OS default.",
        )
        return p
    
    
    def main(argv: Optional[List[str]] = None) -> None:
        """
        CLI entrypoint.
    
        Run from repo root:
    
          python -m tools.foundry_threads
    
        Or with options:
    
          python -m tools.foundry_threads --topic my_topic
        """

            sessions_dir=args.sessions_dir,
            backup=(not args.no_backup),
            quiet=args.quiet,
            downloads_dir=args.downloads_dir,
        )
    
    
    if __name__ == "__main__":
        main()
