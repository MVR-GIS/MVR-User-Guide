## Options — Python library stack for PDF → semantic chunks + metadata

Ranked highest-confidence first, based on: provenance richness, ecosystem maturity, platform-agnosticism, and auditability.

---

### Option 1 ⭐ `unstructured` + LangChain `SemanticChunker`
**What it is:** `unstructured` (Unstructured.io) handles PDF parsing and emits richly typed document elements (NarrativeText, Title, Table, Header, Footer, etc.) with per-element metadata. LangChain's `SemanticChunker` groups sentences by embedding similarity, producing semantically coherent chunks.

| | |
|---|---|
| **PDF parsing metadata** | Page number, element type, filename, file path, last-modified date, coordinates/bounding box, languages detected |
| **Chunking strategy** | Embedding-based semantic similarity (cosine distance threshold) |
| **Provenance depth** | ✅ Best-in-class — element-level origin tracking survives chunking |
| **Ecosystem maturity** | High — both libraries are widely documented and actively maintained |
| **Tradeoffs** | `unstructured` has significant optional dependencies (for high-fidelity PDF: `pdfminer`, `pikepdf`, optionally `tesseract` for scanned docs); LangChain is a large dependency but `langchain-text-splitters` can be installed standalone |
| **Output flexibility** | JSONL for Gemini; Parquet for Foundry — both straightforward |
| **Risk** | LangChain API surface changes frequently; pin versions |

---

### Option 2 `docling` (IBM) + `chonkie`
**What it is:** IBM's `docling` is a newer PDF parser that preserves document structure (headings, sections, tables, lists) and emits a rich DoclingDocument schema. `chonkie` is a lightweight, purpose-built chunking library with a `SemanticChunker`.

| | |
|---|---|
| **PDF parsing metadata** | Section headings, page number, bounding boxes, table/figure labels, document hierarchy |
| **Chunking strategy** | Semantic (embedding-based), token-aware |
| **Provenance depth** | ✅ Strong — structural hierarchy (section path) is preserved per chunk |
| **Ecosystem maturity** | Medium — both are actively developed but newer; fewer production case studies than Option 1 |
| **Tradeoffs** | `docling` excels at structured/formatted PDFs (reports, regulations); may underperform on scanned or unstructured PDFs; `chonkie` has less community documentation |
| **Output flexibility** | Same — JSONL + Parquet |
| **Risk** | Smaller community = fewer Stack Overflow answers; API less stable |

---

### Option 3 `PyMuPDF` + custom semantic chunker (`sentence-transformers`)
**What it is:** `PyMuPDF` (fitz) is a fast, low-level PDF parser. You write a custom semantic chunker using `sentence-transformers` cosine similarity — sliding window grouping sentences until similarity drops below threshold.

| | |
|---|---|
| **PDF parsing metadata** | Page number, PDF metadata dict (title, author, subject, creation date, producer), block coordinates |
| **Chunking strategy** | Custom cosine-similarity sentence grouper — fully transparent |
| **Provenance depth** | ✅ You control every metadata field — maximum auditability |
| **Ecosystem maturity** | `PyMuPDF` is very mature; `sentence-transformers` is mature; the custom chunker is your code |
| **Tradeoffs** | Most code to write and maintain; custom logic must be tested and documented; no abstraction layer handles edge cases automatically |
| **Output flexibility** | Same |
| **Risk** | Maintenance burden; edge cases (multi-column PDFs, footnotes) require custom handling |

---

### Option 4 `unstructured` end-to-end (single library)
**What it is:** `unstructured` alone handles both parsing and chunking (`chunk_by_title()` groups elements under their nearest heading; `chunk_elements()` does fixed-size chunking with overlap). No separate chunking library needed.

| | |
|---|---|
| **PDF parsing metadata** | Same as Option 1 |
| **Chunking strategy** | Title-based (structural) or fixed-size with overlap — **not embedding-based semantic chunking** |
| **Provenance depth** | ✅ Same rich element metadata as Option 1 |
| **Tradeoffs** | `chunk_by_title` is structural, not semantic — chunks follow document headings, not embedding similarity. This may be good enough for well-structured reports, but doesn't satisfy "semantic chunking" in the embedding sense. |
| **Output flexibility** | Same |
| **Risk** | If documents lack consistent heading structure (e.g., scanned PDFs, legacy reports), chunk quality degrades |

---

### Option 5 `pdfplumber` + recursive character splitter (LangChain)
**What it is:** `pdfplumber` extracts text and page metadata; LangChain's `RecursiveCharacterTextSplitter` chunks by character count with overlap.

| | |
|---|---|
| **PDF parsing metadata** | Page number, file path, bounding boxes |
| **Chunking strategy** | Fixed-size with overlap — **not semantic** |
| **Provenance depth** | ⚠️ Adequate but not best-in-class — no element typing, no section heading tracking |
| **Tradeoffs** | Simple, fast, minimal dependencies — but fixed-size chunking splits mid-sentence, mid-paragraph; violates the semantic coherence requirement |
| **Output flexibility** | Same |
| **Risk** | Does not meet your stated goal of "semantic chunking best practice" — not recommended given the audit trail requirements |

# Parsing PDFs

## Concepts

### Why parse?
Raw PDFs are not inherently structured in a way that supports AI-driven workflows. Parsing breaks a document into logical elements like text, headings, and tables, enabling downstream tasks such as semantic chunking and metadata enrichment.

### Structured elements vs. raw text
PDFs can appear as a single block of text or as distinct document elements. Parsing ensures that each element (e.g., a header vs. a paragraph) is correctly classified, which is essential for maintaining context in further processing.

### What metadata ensures traceability?
For AI systems in government workflows, traceability is crucial. Metadata such as source file names, page numbers, and bounding box coordinates allows each chunk to be auditable, reviewable, and validatable.

### Key challenges parsing USACE PDFs
- **Variety:** Documents may include a combination of scanned images, different fonts, and mixed layouts.
- **Consistency:** Headers, tables, and footers may look similar across pages, complicating schema standardization.
- **Accuracy:** Missing or incorrect metadata undermines the ability to audit.

---

## Workflow

### Step 1 — Install dependencies
```bash
# Install Python dependencies for parsing
pip install unstructured pdf2image
```

### Step 2 — Discover documents
Place PDFs in a known local folder structure, for example:
```
/data/reports/
├── 2026/
│   ├── report1.pdf
│   └── design/
│       ├── annex1.pdf
│       └── annex2.pdf
└── 2025/
    ├── report-archive/
    └── ...
```

### Step 3 — Parse PDFs to structured elements
Using the `unstructured` library, process PDFs like this:

```python
from unstructured.partition.pdf import partition_pdf
from pathlib import Path

# Folder path: Replace with the PDF source folder
source_dir = Path("/data/reports")

# Discover and parse PDFs
for pdf_path in source_dir.rglob("*.pdf"):
    elements = partition_pdf(filename=str(pdf_path))

    # Print metadata for inspection
    for element in elements:
        print(element.to_dict())
```

### Step 4 — Verify element correctness / metadata completeness
Inspect parsed output to ensure:
- Every item has **metadata** like `page_number`, `type`, and `layout`.
- Common pitfalls: Elements missing headers or disordered on the page.

---

### Required fields
| Field          | Description                                   | Source              |
|----------------|-----------------------------------------------|---------------------|
| `page_number`  | Page where the element was found             | Extracted metadata  |
| `element_type` | Type of element (e.g., NarrativeText, Title)  | Parsed automatically |
| `coordinates`  | Bounding box for visual elements             | Visual layout map   |
| `source_file`  | File path where the document originated      | Manual provenance   |

---

## Verification Checklist
✅ PDF parsed into robust structured elements.

✅ Metadata fields are consistently populated and complete.

✅ Common edge cases (tables, footnotes) verified manually.

## Common Issues (Troubleshooting)

- Some PDFs may render text incorrectly, causing bounding boxes to fail. Run `pdf2image` manually to create render images and repeat parsing.

### Issue: Element order out-of-sequence
- Extracted elements may sometimes appear out-of-order. Use the `sort_elements()` method or check the reading order manually.

# Parsing PDFs for AI Workflows

## Concepts

### Why we parse PDFs (not just “extract text”)
For enterprise AI-assisted workflows (e.g., **RAG** — Retrieval-Augmented Generation, also called **AI-assisted document retrieval** or an **enterprise knowledge base**), we typically *retrieve* small, relevant portions of documents instead of sending whole documents into a model.

If we only extract raw text, we lose important context that auditors and reviewers need:

- **Where did this statement come from?** (document identity, page number, section)
- **What is the original source?** (file path, file hash, last modified time)
- **What kind of content is it?** (title, paragraph, header/footer, table)
- **Can we reconstruct the evidence trail?** (traceability from chunk → element → PDF)

Parsing is the step where we convert PDFs into **structured elements** (text blocks + rich metadata) so downstream chunking can preserve **source provenance**.

### What we mean by “elements”
An *element* is a unit of extracted content that retains document-aware structure. Depending on the PDF, elements may include:

- titles / headings
- narrative paragraphs
- lists / bullets
- tables (sometimes)
- headers / footers (often noise, but still identifiable)

In this workflow, elements are the **atomic units** we later group into **semantic chunks**.

### Provenance and audit trail requirements (USACE context)
In a government review environment, the outputs of AI-assisted workflows must be defensible. That means every downstream record (chunks, embeddings, search hits, citations) must retain enough metadata to support:

- **traceability:** “show me the exact PDF and page this came from”
- **review:** “is this claim supported by the source?”
- **validation:** “did we ingest the correct version of the document?”
- **auditability:** “what processing steps were applied, by what tool versions, and when?”

**Guiding principle:** If two documents have identical text but different origins (different file, version, date), they must not be treated as the same evidence.

---

## Workflow overview

We will:

1. **Discover** PDFs in nested folders on a local filesystem.
2. **Capture file-level provenance** (path, size, modified time, hashes).
3. **Parse** PDFs into structured elements using `unstructured`.
4. **Persist** parsed outputs to a platform-neutral format (JSONL), preserving metadata.
5. **Verify** parsing quality and metadata completeness before chunking.

---

## Prerequisites

### Execution environment
- Python 3.10+ recommended (pin a version for reproducibility).
- Offline-friendly installation (no cloud APIs required for parsing).

### Python packages (recommended baseline)
We recommend isolating this workflow in a virtual environment and pinning versions.

Core packages:
- `unstructured[pdf]` (PDF parsing)
- `pandas` (inspection / QA)
- `pyarrow` (optional now; useful later for Parquet)
- `pydantic` (optional; schema validation for metadata objects)

> Note: `unstructured`’s PDF support can pull in multiple dependencies depending on your desired fidelity. Start with the minimal set and only add OCR/table dependencies if required by your PDFs.

## Step 1 — Discover PDFs in nested folders

### Goal
Create a deterministic inventory of PDFs to be parsed. This inventory is also your **ingestion manifest** for review.

### Recommendation
Treat each ingestion as a *run* with an explicit run identifier (date + operator + purpose). Store outputs under a run folder such as:

- `runs/2026-04-22_ingest-civilworks-reports/`

This supports reproducibility and auditing.

### Suggested manifest fields (file-level provenance)
At minimum capture:

- `source_path` (relative or absolute; be consistent)
- `source_filename`
- `source_bytes`
- `source_modified_time_utc`
- `sha256` (content hash; critical for version traceability)
- `run_id`
- `collected_at_utc`

---

## Step 2 — Capture file hashes (non-negotiable for traceability)

### Why SHA-256 matters
File paths and modified times are not sufficient for provenance:
- paths can change (moves/renames)
- modified times can be lost or rewritten during copy
- duplicate filenames are common

A cryptographic hash lets you answer:
- “Did we ingest the same file?”
- “Which version of the report was used?”

### Policy recommendation
- Hash the **raw PDF bytes** before parsing.
- Keep the hash associated with *every* downstream record (elements, chunks).

---

## Step 3 — Parse PDFs into elements with `unstructured`

### Goal
Convert each PDF into a list of elements where each element includes:
- extracted text
- element type/category
- page-level pointers (where possible)
- document/file-level provenance (path, sha256, etc.)

### What “good parsing” looks like
- Titles are identified as titles (not mixed into paragraph text).
- Headers/footers are separable (so they can be excluded later).
- Page numbers are present for the majority of narrative content.
- Text is not duplicated (common failure mode with some PDFs).

---

## Step 4 — Persist parsed elements (platform-neutral)

### Recommendation: JSONL as an intermediate artifact
Write one JSON object per element (JSON Lines / JSONL). Reasons:
- stream-friendly
- diff-able for review
- easy to load into Python, Foundry, or other tools
- compatible with later transformations

**Do not** proceed to semantic chunking until the parse artifacts are reviewable.

---

## Step 5 — Verification checklist (release blockers)

Before chunking, verify:

### Parsing validity
- [ ] Every PDF in the manifest produced at least one narrative element.
- [ ] Element text is not empty for the majority of elements.
- [ ] No obvious duplication loops (same paragraph repeated many times).

### Provenance completeness (minimum)
- [ ] Every element includes `source_path` and `sha256`.
- [ ] Every element includes a stable `document_id` (recommend: derived from `sha256`).
- [ ] Page number is present when the parser can provide it (document limitations noted if not).

### Sampling-based review (recommended)
- [ ] Randomly sample N documents and compare extracted text against the PDF visually.
- [ ] Randomly sample elements and verify the element type is plausible (e.g., titles are titles).

For each run:

- `manifest.csv` (file inventory + provenance)
- `elements.jsonl` (parsed elements + metadata)
- `run-metadata.json` (tool versions, environment info, operator, timestamp)

These artifacts form the audit trail and support downstream repeatability.

```python
# Save this as scripts/parse_pdfs_unstructured.py (recommended) and run:
#   python scripts/parse_pdfs_unstructured.py
#
# Or copy/paste into a notebook for an initial trial.

from __future__ import annotations

import csv
import datetime as dt
import hashlib
import json
import os
import platform
import sys
from dataclasses import dataclass
from pathlib import Path
from typing import Any, Dict, Iterable, List, Optional

import pandas as pd
from tqdm import tqdm

# unstructured
from unstructured.partition.pdf import partition_pdf


# ---------------------------
# Configuration (edit these)
# ---------------------------

# Root folder containing PDFs in nested directories
SOURCE_ROOT = Path(r"CHANGE_ME\path\to\pdfs").expanduser().resolve()

# Where outputs will be written
OUTPUT_ROOT = Path("runs").resolve()

# Give each run a stable identifier (recommend: YYYY-MM-DD + short purpose)
RUN_ID = f"{dt.datetime.utcnow().date().isoformat()}_pdf-parse"

# Controls
INCLUDE_HIDDEN_FILES = False
MAX_PDFS: Optional[int] = None  # set e.g. 50 for a quick test, else None


# ---------------------------
# Provenance helpers
# ---------------------------

def utc_now_iso() -> str:
    return dt.datetime.utcnow().replace(tzinfo=dt.timezone.utc).isoformat()

def sha256_file(path: Path, chunk_size: int = 1024 * 1024) -> str:
    """Compute SHA-256 over raw bytes (critical for version traceability)."""
    h = hashlib.sha256()
    with path.open("rb") as f:
        while True:
            b = f.read(chunk_size)
            if not b:
                break
            h.update(b)
    return h.hexdigest()

def is_hidden(path: Path) -> bool:
    # Cross-platform-ish: dotfiles + Windows hidden attribute (best-effort)
    if path.name.startswith("."):
        return True
    if os.name == "nt":
        try:
            import ctypes  # noqa: WPS433
            attrs = ctypes.windll.kernel32.GetFileAttributesW(str(path))
            if attrs == -1:
                return False
            return bool(attrs & 2)  # FILE_ATTRIBUTE_HIDDEN
        except Exception:
            return False
    return False

def discover_pdfs(root: Path) -> List[Path]:
    pdfs: List[Path] = []
    for p in root.rglob("*.pdf"):
        if not INCLUDE_HIDDEN_FILES and is_hidden(p):
            continue
        pdfs.append(p)
    pdfs.sort()
    if MAX_PDFS is not None:
        pdfs = pdfs[:MAX_PDFS]
    return pdfs


# ---------------------------
# Data models (lightweight)
# ---------------------------

@dataclass(frozen=True)
class FileManifestRow:
    run_id: str
    collected_at_utc: str
    source_root: str
    source_path: str
    source_relpath: str
    source_filename: str
    source_bytes: int
    source_modified_time_utc: str
    sha256: str

def file_manifest_row(path: Path, source_root: Path, run_id: str) -> FileManifestRow:
    st = path.stat()
    # Convert mtime to UTC ISO
    mtime = dt.datetime.fromtimestamp(st.st_mtime, tz=dt.timezone.utc).isoformat()
    return FileManifestRow(
        run_id=run_id,
        collected_at_utc=utc_now_iso(),
        source_root=str(source_root),
        source_path=str(path),
        source_relpath=str(path.relative_to(source_root)),
        source_filename=path.name,
        source_bytes=int(st.st_size),
        source_modified_time_utc=mtime,
        sha256=sha256_file(path),
    )


def element_to_record(
    element: Any,
    *,
    run_id: str,
    document_sha256: str,
    source_path: str,
    source_relpath: str,
) -> Dict[str, Any]:
    """
    Convert an unstructured element to a JSON-serializable record with strong provenance.
    """
    text = getattr(element, "text", None)
    category = getattr(element, "category", None)  # sometimes present
    element_type = element.__class__.__name__

    md = getattr(element, "metadata", None)
    md_dict: Dict[str, Any] = {}
    if md is not None:
        # metadata is a dataclass-like object with many optional fields
        # We'll extract a conservative subset + keep the full metadata as a dict where possible.
        # Use vars() as best-effort; if it fails, fallback to string representation.
        try:
            md_dict = {k: v for k, v in vars(md).items() if v is not None}
        except Exception:
            md_dict = {"_metadata_repr": repr(md)}

    # Attempt to capture page numbers when present
    page_number = md_dict.get("page_number") or md_dict.get("page_number_start")

    # Stable document ID recommendation: sha256 (or a prefix)
    document_id = f"sha256:{document_sha256}"

    return {
        # Run/audit fields
        "run_id": run_id,
        "parsed_at_utc": utc_now_iso(),

        # File-level provenance
        "document_id": document_id,
        "document_sha256": document_sha256,
        "source_path": source_path,
        "source_relpath": source_relpath,

        # Element identity
        "element_index": None,  # filled later in order
        "element_type": element_type,
        "element_category": category,

        # Location pointers (best-effort)
        "page_number": page_number,

        # Content
        "text": text,

        # Raw extracted metadata snapshot (keep it; do not discard evidence)
        "unstructured_metadata": md_dict,
    }


# ---------------------------
# Main routine
# ---------------------------

def main() -> None:
    if not SOURCE_ROOT.exists():
        raise FileNotFoundError(f"SOURCE_ROOT does not exist: {SOURCE_ROOT}")

    run_dir = OUTPUT_ROOT / RUN_ID
    run_dir.mkdir(parents=True, exist_ok=False)

    pdf_paths = discover_pdfs(SOURCE_ROOT)
    if not pdf_paths:
        raise RuntimeError(f"No PDFs found under: {SOURCE_ROOT}")

    # Write run metadata (audit trail)
    run_metadata = {
        "run_id": RUN_ID,
        "created_at_utc": utc_now_iso(),
        "source_root": str(SOURCE_ROOT),
        "output_dir": str(run_dir),
        "python": sys.version,
        "platform": {
            "system": platform.system(),
            "release": platform.release(),
            "version": platform.version(),
            "machine": platform.machine(),
        },
        "packages": {
            # Best-effort version capture
            "unstructured": _safe_import_version("unstructured"),
            "pandas": _safe_import_version("pandas"),
            "tqdm": _safe_import_version("tqdm"),
        },
        "assumptions": [
            "PDFs that were scanned have already been OCR'd prior to ingestion.",
            "No cloud APIs are required for parsing.",
        ],
    }
    (run_dir / "run-metadata.json").write_text(json.dumps(run_metadata, indent=2), encoding="utf-8")

    # Build manifest
    manifest_rows: List[FileManifestRow] = []
    for p in tqdm(pdf_paths, desc="Hashing PDFs"):
        manifest_rows.append(file_manifest_row(p, SOURCE_ROOT, RUN_ID))

    manifest_df = pd.DataFrame([r.__dict__ for r in manifest_rows])
    manifest_path = run_dir / "manifest.csv"
    manifest_df.to_csv(manifest_path, index=False)

    # Parse PDFs -> elements.jsonl
    elements_path = run_dir / "elements.jsonl"
    with elements_path.open("w", encoding="utf-8") as f:
        for row in tqdm(manifest_rows, desc="Parsing PDFs"):
            pdf_path = Path(row.source_path)

            # Parse with unstructured
            elements = partition_pdf(
                filename=str(pdf_path),
                # We are NOT doing OCR here; docs should already be OCR'd if scanned.
                # Keep config minimal to reduce environment complexity.
            )

            # Convert to records with provenance
            records = [
                element_to_record(
                    el,
                    run_id=RUN_ID,
                    document_sha256=row.sha256,
                    source_path=row.source_path,
                    source_relpath=row.source_relpath,
                )
                for el in elements
            ]

            # Fill element_index deterministically
            for i, rec in enumerate(records):
                rec["element_index"] = i

            # Write JSONL
            for rec in records:
                f.write(json.dumps(rec, ensure_ascii=False) + "\n")

    print(f"Done.\nRun directory: {run_dir}\nManifest: {manifest_path}\nElements: {elements_path}")


def _safe_import_version(pkg: str) -> Optional[str]:
    try:
        mod = __import__(pkg)
        return getattr(mod, "__version__", None)
    except Exception:
        return None


if __name__ == "__main__":
    main()
```

- **What is the original source?** (file path, file hash, last modified time)
- **What kind of content is it?** (title, paragraph, header/footer, table)
- **Can we reconstruct the evidence trail?** (traceability from chunk → element → PDF)

Parsing is the step where we convert PDFs into **structured elements** (text blocks + rich metadata) so downstream chunking can preserve **source provenance**.

### What we mean by “elements”
An *element* is a unit of extracted content that retains document-aware structure. Depending on the PDF, elements may include:

- titles / headings
- narrative paragraphs
- lists / bullets
- tables (sometimes)
- headers / footers (often noise, but still identifiable)

In this workflow, elements are the **atomic units** we later group into **semantic chunks**.

### Provenance and audit trail requirements (USACE context)
In a government review environment, AI-assisted outputs must be defensible. That means every downstream record (chunks, embeddings, search hits, citations) must retain enough metadata to support:

- **traceability:** “show me the exact PDF and page this came from”
- **review:** “is this claim supported by the source?”
- **validation:** “did we ingest the correct version of the document?”
- **auditability:** “what processing steps were applied, by what tool versions, and when?”

**Guiding principle:** If two documents have identical text but different origins (different file, version, date), they must not be treated as the same evidence.

---

## Workflow overview

We will:

1. **Discover** PDFs in nested folders on a local filesystem.
2. **Capture file-level provenance** (path, size, modified time, hashes).
3. **Parse** PDFs into structured elements using `unstructured`.
4. **Persist** parsed outputs to a platform-neutral format (JSONL), preserving metadata.
5. **Verify** parsing quality and metadata completeness before chunking.

## Runnable script: PDF inventory + parsing + JSONL outputs

### What this script produces
For each run, it creates a run folder with:

- `manifest.csv` — one row per PDF (file-level provenance)
- `elements.jsonl` — one JSON object per extracted element (element-level provenance)
- `run-metadata.json` — run info + tool versions (audit trail)

### How to run
- Set `SOURCE_ROOT` to the folder containing PDFs (nested folders OK).
- Run the script.
- Inspect outputs and perform verification checks below.

    # Save this as scripts/parse_pdfs_unstructured.py (recommended) and run:
    #   python scripts/parse_pdfs_unstructured.py
    #
    # Or copy/paste into a notebook for an initial trial.

    from __future__ import annotations

    import datetime as dt
    import hashlib
    import json
    import os
    import platform
    import sys
    from dataclasses import dataclass
    from pathlib import Path
    from typing import Any, Dict, List, Optional

    import pandas as pd
    from tqdm import tqdm

    from unstructured.partition.pdf import partition_pdf


    # ---------------------------
    # Configuration (edit these)
    # ---------------------------

    # Root folder containing PDFs in nested directories
    SOURCE_ROOT = Path(r"CHANGE_ME\path\to\pdfs").expanduser().resolve()

    # Where outputs will be written
    OUTPUT_ROOT = Path("runs").resolve()

    # Give each run a stable identifier (recommend: YYYY-MM-DD + short purpose)
    RUN_ID = f"{dt.datetime.utcnow().date().isoformat()}_pdf-parse"

    # Controls
    INCLUDE_HIDDEN_FILES = False
    MAX_PDFS: Optional[int] = None  # set e.g. 50 for a quick test, else None


    # ---------------------------
    # Provenance helpers
    # ---------------------------

    def utc_now_iso() -> str:
        return dt.datetime.utcnow().replace(tzinfo=dt.timezone.utc).isoformat()

    def sha256_file(path: Path, chunk_size: int = 1024 * 1024) -> str:
        """Compute SHA-256 over raw bytes (critical for version traceability)."""
        h = hashlib.sha256()
        with path.open("rb") as f:
            while True:
                b = f.read(chunk_size)
                if not b:
                    break
                h.update(b)
        return h.hexdigest()

    def is_hidden(path: Path) -> bool:
        # Cross-platform-ish: dotfiles + Windows hidden attribute (best-effort)
        if path.name.startswith("."):
            return True
        if os.name == "nt":
            try:
                import ctypes
                attrs = ctypes.windll.kernel32.GetFileAttributesW(str(path))
                if attrs == -1:
                    return False
                return bool(attrs & 2)  # FILE_ATTRIBUTE_HIDDEN
            except Exception:
                return False
        return False

    def discover_pdfs(root: Path) -> List[Path]:
        pdfs: List[Path] = []
        for p in root.rglob("*.pdf"):
            if not INCLUDE_HIDDEN_FILES and is_hidden(p):
                continue
            pdfs.append(p)
        pdfs.sort()
        if MAX_PDFS is not None:
            pdfs = pdfs[:MAX_PDFS]
        return pdfs


    # ---------------------------
    # Data models (lightweight)
    # ---------------------------

    @dataclass(frozen=True)
    class FileManifestRow:
        run_id: str
        collected_at_utc: str
        source_root: str
        source_path: str
        source_relpath: str
        source_filename: str
        source_bytes: int
        source_modified_time_utc: str
        sha256: str

    def file_manifest_row(path: Path, source_root: Path, run_id: str) -> FileManifestRow:
        st = path.stat()
        mtime = dt.datetime.fromtimestamp(st.st_mtime, tz=dt.timezone.utc).isoformat()
        return FileManifestRow(
            run_id=run_id,
            collected_at_utc=utc_now_iso(),
            source_root=str(source_root),
            source_path=str(path),
            source_relpath=str(path.relative_to(source_root)),
            source_filename=path.name,
            source_bytes=int(st.st_size),
            source_modified_time_utc=mtime,
            sha256=sha256_file(path),
        )

    def element_to_record(
        element: Any,
        *,
        run_id: str,
        document_sha256: str,
        source_path: str,
        source_relpath: str,
    ) -> Dict[str, Any]:
        text = getattr(element, "text", None)
        category = getattr(element, "category", None)
        element_type = element.__class__.__name__

        md = getattr(element, "metadata", None)
        md_dict: Dict[str, Any] = {}
        if md is not None:
            try:
                md_dict = {k: v for k, v in vars(md).items() if v is not None}
            except Exception:
                md_dict = {"_metadata_repr": repr(md)}

        page_number = md_dict.get("page_number") or md_dict.get("page_number_start")

        document_id = f"sha256:{document_sha256}"

        return {
            "run_id": run_id,
            "parsed_at_utc": utc_now_iso(),

            "document_id": document_id,
            "document_sha256": document_sha256,
            "source_path": source_path,
            "source_relpath": source_relpath,

            "element_index": None,
            "element_type": element_type,
            "element_category": category,

            "page_number": page_number,

            "text": text,

            "unstructured_metadata": md_dict,
        }


    # ---------------------------
    # Main routine
    # ---------------------------

    def _safe_import_version(pkg: str) -> Optional[str]:
        try:
            mod = __import__(pkg)
            return getattr(mod, "__version__", None)
        except Exception:
            return None

    def main() -> None:
        if not SOURCE_ROOT.exists():
            raise FileNotFoundError(f"SOURCE_ROOT does not exist: {SOURCE_ROOT}")

        run_dir = OUTPUT_ROOT / RUN_ID
        run_dir.mkdir(parents=True, exist_ok=False)

        pdf_paths = discover_pdfs(SOURCE_ROOT)
        if not pdf_paths:
            raise RuntimeError(f"No PDFs found under: {SOURCE_ROOT}")

        run_metadata = {
            "run_id": RUN_ID,
            "created_at_utc": utc_now_iso(),
            "source_root": str(SOURCE_ROOT),
            "output_dir": str(run_dir),
            "python": sys.version,
            "platform": {
                "system": platform.system(),
                "release": platform.release(),
                "version": platform.version(),
                "machine": platform.machine(),
            },
            "packages": {
                "unstructured": _safe_import_version("unstructured"),
                "pandas": _safe_import_version("pandas"),
                "tqdm": _safe_import_version("tqdm"),
            },
            "assumptions": [
                "PDFs that were scanned have already been OCR'd prior to ingestion.",
                "No cloud APIs are required for parsing.",
            ],
        }
        (run_dir / "run-metadata.json").write_text(json.dumps(run_metadata, indent=2), encoding="utf-8")

        manifest_rows: List[FileManifestRow] = []
        for p in tqdm(pdf_paths, desc="Hashing PDFs"):
            manifest_rows.append(file_manifest_row(p, SOURCE_ROOT, RUN_ID))

        manifest_df = pd.DataFrame([r.__dict__ for r in manifest_rows])
        manifest_path = run_dir / "manifest.csv"
        manifest_df.to_csv(manifest_path, index=False)

        elements_path = run_dir / "elements.jsonl"
        with elements_path.open("w", encoding="utf-8") as f:
            for row in tqdm(manifest_rows, desc="Parsing PDFs"):
                pdf_path = Path(row.source_path)

                elements = partition_pdf(
                    filename=str(pdf_path),
                )

                records = [
                    element_to_record(
                        el,
                        run_id=RUN_ID,
                        document_sha256=row.sha256,
                        source_path=row.source_path,
                        source_relpath=row.source_relpath,
                    )
                    for el in elements
                ]

                for i, rec in enumerate(records):
                    rec["element_index"] = i

                for rec in records:
                    f.write(json.dumps(rec, ensure_ascii=False) + "\n")

        print(f"Done.\nRun directory: {run_dir}\nManifest: {manifest_path}\nElements: {elements_path}")

    if __name__ == "__main__":
        main()

## Verification checklist (use after running the script)

### Parse validity
- [ ] `manifest.csv` includes every PDF you intended to ingest (spot-check folder coverage).
- [ ] `elements.jsonl` is non-empty.
- [ ] A sample of documents produces narrative elements (not just headers/footers).

- [ ] Every element JSON object has: `document_sha256`, `source_relpath`, `element_index`, `text` (or a documented reason if `text` is empty).

- [ ] Randomly sample 5 PDFs, open them, and verify a few extracted elements match the page content.
